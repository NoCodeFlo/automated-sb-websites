Intro sentence
Build a modern one-page website for “Kräuter Atelier Luzern” that preserves the brand identity (logo, earthy visuals, voice), improves clarity and accessibility, and consolidates all content into a fast, mobile-first landing page.

Project Overview
- Current site audit (from provided HTML):
  - Pages/links in header: Workshops, Kräuterspaziergänge, Termine, Atelier, Über mich, Kontakt.
  - Hero: Full-width cover with parallax background (flowers) and centered white logo.
  - Content blocks: Welcome headline and paragraph; Gutscheine block (with gift card image); notice about “Termine 2025” being available; footer with address and email.
  - Visual identity: Black/white plant logo; photography with natural, earthy tones; dark footer (#111111); body/link color tending to deep brown (#2a2623).
  - Contact info: Dreilindenstrasse 46, 6006 Luzern; info@kraeuter-atelier.com.
  - SEO meta title: “Heilpflanzen und Wildkräuter Kurse - Kräuter Atelier Luzern”; meta description present.
  - CMS: WordPress; theme: Appleton (child of Blockbase); Jetpack; Yoast SEO.
- Content summary:
  - Name: Kräuter Atelier Luzern
  - Slogan/tagline in context: “Heilpflanzen und Wildkräuter Kurse”
  - Services: Workshops/Kurse; Kräuterspaziergänge; Termine/Programm; Gutscheine; About/Atelier; Contact.
  - Core text: Welcome paragraph about rediscovering traditional plant knowledge; instruction to order vouchers via contact form.
- Goals for the new one-page:
  - Keep identity: use existing logo, photography, earthy palette, tone of voice.
  - Modernize: clear information architecture, stronger CTAs, lightweight performance, accessible UI, mobile-first layout.
  - Consolidate: collapse subpages into sections with anchors.

Tech & Approach
- Stack: Static one-page site using:
  - Option A: Next.js 14 App Router + Tailwind CSS + next-seo.
  - Option B: Astro + Tailwind CSS for even lighter output.
  - Either option: ship semantic HTML, zero-JS where possible; hydrate only for nav, accordions, lightbox, and contact form.
- Performance: Responsive images (srcset/sizes); lazy-load below the fold; preconnect to image CDN; use CSS prefers-reduced-motion; minimal JS; CSS variables for theming.
- Forms: Use a simple serverless form handler (Netlify Forms/Formspark) or mailto fallback.
- Deployment: Static hosting (Vercel/Netlify); include redirects from legacy slugs to anchors.

Accessibility specifics
- Semantic headings: single H1 on hero; H2 for sections.
- Focus states: visible outline; focus-trap in mobile menu; skip-to-content link.
- Color contrast: verify all text on hero overlay and buttons >= 4.5:1.
- Motion: disable parallax/animations when prefers-reduced-motion is set.
- Images: descriptive alt text; decorative images with empty alt.
- Forms: label fields; aria-live for success/error messages; prevent placeholder-only.
